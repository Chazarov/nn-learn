# Визуализация карты Кохонена

Формулы в разметке `$…$` / `$$…$$`: в VSCode для предпросмотра LaTeX обычно нужно расширение (например *Markdown+Math*).

Модуль строит изображения по матрице весов `weights` размера `(rows·cols) × dim`: строка с индексом `i·cols + j` соответствует узлу сетки с координатами `(i, j)`.

---

## Гексагональная сетка (U-matrix)

**Назначение.** Плотная упаковка шестиугольников (ориентация *pointy-top*, схема смещения *odd-r*) задаёт только **геометрию отрисовки**; расчёт U-matrix по-прежнему использует **прямоугольную** решётку `rows × cols` и **четырёхсвязное** соседство (верх / низ / лево / право).

**Алгоритм размещения центров.** Задаётся радиус описанной окружности шестиугольника $R$ (расстояние от центра до вершины). Шаги между центрами соседних ячеек в одной строке и между строками:

$$
\Delta x = \sqrt{3}\,R,\qquad
\Delta y = \frac{3}{2}\,R,\qquad
s = \frac{\sqrt{3}}{2}\,R = \frac{\Delta x}{2}.
$$

где:  
$R$ — параметр `hex_radius` (пиксели);  
$\Delta x$ — горизонтальное расстояние между центрами в строке;  
$\Delta y$ — вертикальное расстояние между центрами в соседних строках;  
$s$ — дополнительный горизонтальный сдвиг для строк с нечётным индексом $i$.

Координаты центра ячейки $(i,j)$ (начало координат — левый верх изображения, ось $y$ вниз; `pad` — поле):

$$
c_x = \mathrm{pad} + \frac{\sqrt{3}}{2}R + j\,\Delta x + (i \bmod 2)\,s,\qquad
c_y = \mathrm{pad} + R + i\,\Delta y.
$$

где:  
$i \in \{0,\ldots,\mathrm{rows}-1\}$, $j \in \{0,\ldots,\mathrm{cols}-1\}$;  
слагаемое $(i \bmod 2)\,s$ задаёт «шахматный» сдвиг нечётных строк.

**Вершины шестиугольника.** Шесть вершин на окружности радиуса $R$ с центром $(c_x, c_y)$; полярный угол $k$-й вершины (в радианах):

$$
\varphi_k = -\frac{\pi}{2} + k\frac{\pi}{3},\quad k=0,\ldots,5.
$$

где:  
первая вершина направлена «вверх» по экрану (меньше координата $y$);  
декартовы координаты: $x_k = c_x + R\cos\varphi_k$, $y_k = c_y + R\sin\varphi_k$.

**Размер изображения.** Ширина и высота — по максимальным $c_x, c_y$ с учётом вылета гекса за центр ($\pm \frac{\sqrt{3}}{2}R$ по горизонтали, $+R$ по вертикали вниз) и полей `pad`. Итого линейный масштаб картинки определяется **`rows`, `cols`, `hex_radius`, `pad`**; при большей сетке или большем $R$ изображение больше.

---

## Значение каждой ячейки

**Один шестиугольник = один нейрон карты** (один узел решётки $(i,j)$). Его **весовой вектор** — это строка `weights[indices[i, j], :]` в коде; отдельно «нейрон без веса» не существует: узел карты задаётся именно этой строкой.

---

## U-matrix: расчёт величины в ячейке

Для узла $(i,j)$ собираются все существующие соседи по четырём направлениям; для каждого соседа $(i',j')$ считается евклидово расстояние между весовыми векторами узлов в пространстве признаков:

$$
d\bigl((i,j),(i',j')\bigr) = \bigl\|\mathbf{w}_{i,j} - \mathbf{w}_{i',j'}\bigr\|_2.
$$

где:  
$\mathbf{w}_{i,j}$ — вектор весов нейрона в позиции $(i,j)$ (строка матрицы `weights`).

Значение U-matrix в ячейке — **среднее** по всем таким расстояниям к соседям (у краевых узлов соседей меньше четырёх):

$$
U_{i,j} = \frac{1}{|N_{i,j}|} \sum_{(i',j') \in N_{i,j}} d\bigl((i,j),(i',j')\bigr).
$$

где:  
$N_{i,j}$ — множество индексов соседей на решётке `rows × cols` с 4-связностью.

Для раскраски выполняется линейная нормировка всех $U_{i,j}$ на отрезок $[0,255]$ по минимуму и максимуму по всей сетке (если разброс нулевой — подставляется константа 128).

---

## Цвета

И для U-matrix, и для **component plane** используется палитра **Viridis** (`cv2.COLORMAP_VIRIDIS`): скаляр в $[0,255]$ после нормировки переводится в BGR-тройку. **Низкие** значения нормированной величины — один конец спектра палитры, **высокие** — другой (интерпретация «холодный / тёплый» смысл задаёт именно Viridis, не произвольная шкала).

- **U-matrix:** цвет кодирует среднюю «разрывность» весов между соседями на карте (типично: **больше** $U$ → сильнее отличаются соседние нейроны в исходном пространстве признаков).
- **Component plane:** в ячейку попадает одна компонента вектора веса `input_id` для всех нейронов; после нормировки по минимуму/максимуму на всей сетке — та же Viridis. **Каждый квадрат** на промежуточной решётке — один нейрон; перед colormap поле увеличивается `resize` с `INTER_NEAREST`.
