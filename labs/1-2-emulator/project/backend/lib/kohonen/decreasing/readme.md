# Линейное расписание скорости обучения SOM

Для дискретных шагов обучения $k = 0, 1, \ldots, K-1$ задаётся линейная интерполяция между начальной и конечной скоростью.

## Формула

При $K \ge 2$:

$$
\alpha(k) = \alpha_{\mathrm{start}} + (\alpha_{\mathrm{end}} - \alpha_{\mathrm{start}}) \cdot \frac{k}{K - 1}.
$$

При $K = 1$ (один шаг) полагают $\alpha(0) = \alpha_{\mathrm{start}}$.

## Где

- $k$ — индекс текущего шага обучения (в коде: аргумент ``step_index``, после приведения к диапазону $[0, K-1]$);
- $K$ — общее число шагов (аргумент ``total_steps``);
- $\alpha_{\mathrm{start}}$ — скорость в начале обучения (аргумент ``rate_start``);
- $\alpha_{\mathrm{end}}$ — скорость в конце обучения (аргумент ``rate_end``);
- $\alpha(k)$ — скорость на шаге $k$ (возвращаемое значение функции ``decreasing_linear_rate``).

Для ширины окрестности $\sigma(k)$ на той же сетке шагов $k = 0, \ldots, K-1$ используется та же линейная формула, функция ``decreasing_linear_sigma``; требуется $0 < \sigma_{\mathrm{end}} < \sigma_{\mathrm{start}}$.

## Кратко

Коэффициент $\frac{k}{K-1}$ пробегает от $0$ до $1$, поэтому $\alpha(k)$ равномерно переходит от $\alpha_{\mathrm{start}}$ к $\alpha_{\mathrm{end}}$. Для **убывающего** расписания нужно $\alpha_{\mathrm{start}} > \alpha_{\mathrm{end}}$. Диапазон $(0, 1]$ согласован с проверкой скорости в ``WeightApdator``.
